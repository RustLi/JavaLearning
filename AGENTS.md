# AI 工作规范（必须严格遵守）

## 1. 任务来源
- 所有开发任务必须来自：
  - TASKS.md（具体可执行任务）
- 任务拆分必须使用 Shrimp Task Manager（MCP）。

## 2. 任务执行流程（强制）
1. 执行前必须读取 TASKS.md / Shrimp 任务列表。
2. 每次只处理一个任务。
3. 完成任务后必须：
   - 调用 Shrimp 标记为 done
   - 同步更新 TASKS.md，将任务标记为 ✅ 已完成
4. 不允许重复执行已完成任务。

## 3. Shrimp 协作规则（最高优先级）
- 所有任务规划必须通过：`plan_task`
- 所有任务执行必须通过：`execute_task`
- 所有任务完成必须通过：`mark_task_done`
- 任何状态变更必须同步写入TASKS.md，并提交 Git。

## 4. TASKS.md 格式规范
- 待开发：📋
- 进行中：🔨
- 已完成：✅

示例：
## 已完成
- ✅ 创建用户数据库表结构

## 进行中
- 🔨 实现登录接口

## 待开发
- 📋 实现注册接口
- 📋 编写表单验证

## 6. 禁止行为
- 禁止手动修改 TASKS.md 状态（必须通过 Shrimp）
- 禁止跳过任务
- 禁止不更新状态就进入下一项
